# YOLOv8 数据集制作与训练（手把手 / 可直接照做）

这份文档的目标是：在你自己的摄像头场景里，把“分神信号”拆成更客观、可标注、可迭代的目标检测任务，并最终能接入本项目进行实时检测。

本教程采用你指定的类别设计：
- `phone`：手里/桌面出现手机（检测到且持续出现 → 分神加权）
- `head_down`：低头（可选，主观更强，需要一致的标注标准）
- `no_person`：不建议用框标（更适合分类/规则：检测不到人脸/人离开）

> 强烈建议：先只做 `phone`，跑通“采集→标注→训练→验证→接入”的全流程后，再考虑 `head_down`。

---

## 0. 你要做的事（总览）

1) 采集素材：拍视频 → 抽帧成图片（避免相邻帧太像）  
2) 用 labelImg 做框标：导出 YOLO 格式 txt  
3) 整理目录结构 + 写 `data.yaml`  
4) 用 Ultralytics 训练：`yolo detect train ...`  
5) 用几张图片快速验证：`yolo predict ...`  
6) （可选）把权重接入本项目

---

## 1. 类别设计：为什么这样更稳

### 1.1 `phone`（推荐优先）
优点：客观、边界清晰、模型可解释、误判更容易通过补数据修正。

### 1.2 `head_down`（可选）
缺点：主观性强。你必须先写出“什么算低头”的统一标准，否则数据集会“自相矛盾”，模型学不出来。

### 1.3 `no_person`（不建议框标）
原因：这是“场景/状态”，不是“物体”。用 YOLO 框一个 `no_person` 基本没有意义；更推荐：
- 规则：检测不到人脸/人离开视野
- 或分类：整张图做二分类（person / no_person）

本项目当前就有“检测不到人脸 → 视为休息/离开”的逻辑（`no_face`）。

## 2. 标注工具：labelImg（两种方式）

### 方式 A（推荐）：独立安装/独立环境
labelImg 依赖 Qt（常见是 PyQt5），而本项目 GUI 用的是 PySide6。为了避免 Qt 依赖冲突，建议 labelImg 单独装：

- 用独立 venv 安装（最干净）
- 或下载 labelImg 的 Windows 发布包（zip/exe）直接运行

### 方式 B：装在同一个 venv
也能用，但如果你遇到 Qt 相关报错/冲突，优先改回“方式 A”。

---

## 3. 采集素材（图片/视频抽帧）

### 3.1 先定一个最小可用数据量
第一次跑通流程不需要很多，建议：
- `phone`：300–1000 张起步（越多越稳）
- `head_down`：至少再加 300+ 张“明确低头”的样本（并保证标准一致）

### 3.2 素材多样性清单（尽量覆盖）
- 光线：白天 / 夜间 / 背光
- 角度：摄像头高一点 / 低一点 / 侧一点
- 手机形态：黑手机 / 白手机 / 带壳 / 反光 / 屏幕亮暗
- 行为：手持、贴脸、放桌面、半遮挡（手/杯子/书）
- 负样本：桌面上有其他物体但没有手机（遥控器、鼠标、纸巾等）

### 3.3 抽帧建议（关键：减少相邻帧重复）
- 从视频抽成 1fps 或 2fps 的图片，能显著减少“几乎一样的图片”占比。
- 只截取“关键动作片段”也可以（减少无效数据）。

---

## 4. 标注规范（最重要：统一标准）

### 4.1 通用规则
- 框尽量“贴合目标主体”，不要把大量背景带进去
- 模糊/遮挡严重到你自己都不确定的图：宁可不标，后续作为难例再加
- 负样本一定要保留（没有手机的图也要进训练集），否则会“见啥都像手机”

### 4.2 `phone` 标注规则（建议照这个执行）
你每标一张，都按以下规则判断：

1) 框什么：只框住手机本体（机身），不要把整只手框进去  
2) 桌面手机：手机放桌面也要框（因为你定义它也是分神信号）  
3) 多个手机：每个手机单独框一个框  
4) 太小/太糊：手机在画面里小到难辨、或运动模糊严重 → 可以不标（或放难例集）  
5) 反光/屏幕：只要你确定是手机，就框机身；屏幕亮暗不重要

### 4.3 `head_down` 标注规则（可选，但必须先定标准）
建议你先写一个“可操作”的标准，比如：
- “低头”定义为：眼睛明显朝下（看不到眼白/视线方向明显向下），或头部朝下倾斜明显（能稳定判断）
- “非低头”：只是轻微低一点、或在看屏幕下半部但仍明显在看屏幕

框的范围（必须固定一种）建议二选一：
- 方案 1：只框头部（含脸+头发），更稳定  
- 方案 2：框上半身（含头+肩），更容错但噪声更大

一旦选定，不要混用，否则会降低一致性。

### 4.4 `no_person` 不做框标
如果你硬要标，会遇到“框哪里？”这个无法一致回答的问题。建议用规则实现：
- MediaPipe 检测不到脸 → `no_face`（本项目已实现）

## 5. 目录结构（YOLO 标准）

建议按下面结构整理（不要改名乱放）：

```
dataset/
  images/
    train/
    val/
  labels/
    train/
    val/
  data.yaml
```

### 5.1 训练/验证划分建议
- `val` 至少留 10%（或 100–300 张起步）
- `val` 不要包含“几乎相同的连续帧”（否则验证分数虚高）

---

## 6. 用 labelImg 导出 YOLO 格式

1) 打开 labelImg  
2) 设置保存格式为 **YOLO**（YOLO txt）  
3) 选择图片目录（例如 `dataset/images/train`）  
4) 开始标注，保存后会生成同名 `.txt` 到你设置的输出目录  

检查规则：
- 每张图 `xxx.jpg` 应该对应一个 `xxx.txt`
- `.txt` 每一行格式为：`class_id x_center y_center w h`（都是 0~1 的归一化数值）

---

## 7. `data.yaml` 怎么写（示例）

`dataset/data.yaml`（类别顺序要与你在标注工具里一致）：

```yaml
path: dataset
train: images/train
val: images/val
names:
  0: phone
  1: head_down
```

如果你阶段 1 只训练手机，就先只保留 `phone`：

```yaml
path: dataset
train: images/train
val: images/val
names:
  0: phone
```

## 8. 用 Ultralytics 训练（建议从小模型开始）

安装依赖（如果你在本项目虚拟环境里训练，可以跳过，因为 requirements.txt 已包含 ultralytics）：

```powershell
pip install ultralytics
```

训练命令示例（CPU 也能跑，但会慢；先少跑几轮验证流程）：

```powershell
yolo detect train data=dataset/data.yaml model=yolov8n.pt imgsz=640 epochs=30 batch=8
```

训练完成后，一般会生成：
- `runs/detect/train/weights/best.pt`
- `runs/detect/train/weights/last.pt`

---

## 9. 快速验证（强烈建议在接入项目前做）

用几张图片跑一下推理，确认模型真的在你场景下能找到目标：

```powershell
yolo detect predict model=runs/detect/train/weights/best.pt source=dataset/images/val save=True conf=0.35
```

输出通常在 `runs/detect/predict/`，看一眼框得是否合理：
- 漏检多 → 增加同类型数据 + 降低 conf 试试
- 误检多 → 增加负样本（没有手机但“很像手机”的物体）+ 统一标注标准

## 10. 接入本项目（可选）

### 10.1 本项目当前的默认行为
本项目默认用 `yolov8n.pt` 做手机检测，并且会找 COCO 类别名 `cell phone`。

你需要注意两点：
1) 运行主程序时，只有当项目根目录存在 `yolov8n.pt` 才会启用手机检测（避免自动联网下载）  
2) 如果你训练的是自定义数据集且类别名是 `phone`，你需要让代码匹配你的类别名

### 10.2 替换权重
把你训练好的权重放到项目根目录，例如：
- `best.pt` → 改名为 `yolov8n.pt`（最省事）

或者你也可以改代码，让它加载你自定义的文件名（见下）。

### 10.3 如果你自定义类别名是 `phone`
本项目里判断类别名的位置在：
- `daydream_vision.py`
- `app/vision/analyzer.py`

把对 `cell phone` 的判断改为你的类别名（例如 `phone`），并确保权重文件里的 `names` 确实是你期望的名称。

## 11. 常见坑（踩一遍能省很多时间）

- 标注标准不一致：模型会“学不会”，或者泛化很差  
- `val` 里包含相邻帧：验证结果虚高，上线一跑就崩  
- 数据太少就想要很强：先跑通流程再堆数据  
- `head_down` 一开始就做：最容易把流程卡死（因为标准不清晰）  
- labelImg 与本项目同 venv 出现 Qt 冲突：换成“独立 venv/独立安装”即可
