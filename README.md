# Daydream Alert（Windows 本地桌面应用）

你是不是也有这种时刻：

- 计划写得很好，但缺少“强制约束力”，一走神就停不下来  
- 明明很想变强，却总觉得自己“不够努力/不上心”，于是更焦虑、更自责  

**Daydream Alert** 是一个轻便的 Windows 本地桌面应用：用“摄像头 + 当前前台窗口/应用”实时判断你是否在**工作 / 分神 / 休息**，并在分神累计超过阈值时给出**全屏红色警报**，把你温柔但坚定地拉回任务本身。

## 适合人群

- 年轻群体：需要一个可落地的专注辅助工具  
- 那些总觉得自己是因为不努力不上心才没有成功的牛马们和潜力股们  
- 希望“计划能被执行”，而不是只停留在计划表上的人

## 功能概览（v1）

- **本地运行**：默认不保存截图/视频，只记录状态时间段  
- **双通道判定**（可解释）：
  - 前台窗口/进程：按任务配置的白名单判断“是否在任务相关界面”
  - 摄像头：低头/视线粗判（MediaPipe，可关）+ 手机检测（YOLOv8，可关）
- **警报状态机**：
  - 分神累计 ≥ `3min` → 置顶全屏红色警报：`请回到任务本身`
  - 重新进入工作并持续 ≥ `2min` → 自动解除警报
- **退出按钮**：任务开始 `10min` 后才显示（防止“轻易逃跑”）
- **数据记录**：
  - `logs/`：JSON + CSV（每段状态的开始/结束时间、原因、前台窗口信息）
  - `data/daydream_focus.sqlite3`：SQLite 数据库（更适合后续统计/可视化）

## 安装与启动

环境：
- Windows 10/11
- 推荐 Python 3.11+（本项目在 Python 3.13 下通过语法检查与单测）

```powershell
cd D:\daydream
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
pip install -r requirements-dev.txt
python main.py
```

（可选）运行单测：
```powershell
python -m pytest -q
```

## 如何使用

1) 打开应用进入首页  
2) 填写：任务名称、起止时间、阈值（分神触发/解除）  
3) 配置“允许的应用白名单”：
   - **进程名**（推荐先填）：例如 `WINWORD.EXE` / `chrome.exe` / `code.exe`
   - **窗口标题关键词**（可选）：例如 `LeetCode` / `Notion` / `论文`
4) 点击开始进入任务页  
5) 当分神累计超过阈值，会出现置顶全屏红色警报；当持续工作超过解除阈值，自动解除  

提示：
- 手机检测（YOLOv8）默认只在项目根目录存在 `yolov8n.pt` 时启用；否则会自动跳过，避免联网下载权重。
- 如果你觉得电脑性能吃紧，可以在首页关闭“YOLO 手机检测”，只保留窗口白名单 + Face 粗判。

本项目 v1 会把结构化数据写到：
- `data/daydream_focus.sqlite3`（自动创建，已在 `.gitignore` 忽略，不会提交到仓库）

同时为了方便你直接查看/导出，也会写：
- `logs/*.json`
- `logs/*.csv`

> 隐私说明：本项目默认不保存画面、不上传网络；数据库/日志只包含时间段、状态、原因、前台窗口元信息（进程名、窗口标题、pid）。

### 数据库里有什么

- `sessions`：一次任务会话（任务名、开始/结束时间、当次配置 JSON）
- `segments`：会话内的状态片段（work/distracted/rest，每段起止时间、原因、前台窗口信息）

## 为未来“监控/RTSP 接入”预留的接口

当前版本使用本机摄像头；后续要接入监控（例如 RTSP）时，会沿用现有的 `VideoSource` 抽象（在 `app/io/video_source.py`），再实现 `RtspStreamSource(url)` 即可把数据源替换掉（v1 未实现流接入，仅保留扩展点）。
